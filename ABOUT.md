# 关于这个仓库

`Hermes-learn` 是一套中文 Hermes Agent 源码学习笔记。

我更关心的是“Agent 工程到底怎么落地”，而不是只解释几个流行概念。一个真正可用的 Agent，不只是把模型接上工具。它还要处理上下文装配、工具暴露、工具执行失败、会话恢复、长期记忆、技能沉淀、权限控制、外部插件、多平台运行和成本控制。

Hermes 的价值在于，它把这些问题放进了一个完整 runtime 里。读这个项目，可以看到很多 Agent 框架从 demo 走向产品时必须面对的细节。

## 关注的问题

这个仓库会持续围绕这些问题展开：

- 模型每一轮到底看到了什么？
- system prompt 为什么要分层？
- 工具为什么不是一个静态列表？
- 一个 tool call 如何变成一次真实工具执行？
- 工具失败后，Agent 如何继续对话？
- 长期记忆和历史搜索的边界在哪里？
- Skill 是什么，为什么它不等于普通工具？
- SessionDB 为什么重要？
- 上下文压缩什么时候触发？
- 插件和 MCP 如何扩展 Agent 能力？
- Hermes 和 Codex、Claude Code 在同一问题上有哪些不同取舍？

## 写法

笔记按功能组织，而不是按目录组织。

每个功能点通常包含：

- 设计问题
- 源码入口
- 关键源码片段
- 工程实现解释
- 失败路径或边界条件
- 与其他 Agent 框架的对比
- 常见误解

源码片段只保留必要部分。长代码不会直接搬进笔记，读者应该回到源码里继续阅读。

## 读者预期

读完这些笔记，应该能回答的不只是“这个函数在哪里”，而是：

- 为什么 Hermes 要这样设计？
- 如果我自己实现，会在哪些地方踩坑？
- 哪些能力属于 prompt 层，哪些属于 runtime 层？
- 哪些状态应该进入 memory，哪些应该留在 session history？
- 一个 Agent 框架要从玩具走向工程系统，需要补哪些模块？

这套笔记会保持中文表达，尽量写得像老师带读源码，而不是自动生成的摘要。
